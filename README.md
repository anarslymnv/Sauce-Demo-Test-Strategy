# SauceDemo Saytı Üçün Test Strategiyası

## 1. Məqsəd
SauceDemo saytının əsas funksiyalarının düzgün işlədiyindən əmin olmaq üçün testlərin necə aparılacağını müəyyən etmək.

## 2. Test Növləri
- **Funksional test:**  
  Saytın giriş, məhsul əlavə etmə, səbət, sifariş və səhv mesajları kimi əsas funksiyalarının yoxlanılması.  

- **İstifadəçi interfeysi (UI) testi:**  
  Saytın müxtəlif cihaz və brauzerlərdə düzgün və rahat görünməsinin yoxlanılması.  

- **Manual test:**  
  Əsas test ssenarilərinin əl ilə icrası.  

- **Kros-brauzer testi:**  
  Chrome, Firefox, Edge və Safari kimi məşhur brauzerlərdə testlərin aparılması.

## 3. Test Əhatəsi
- **Daxil olanlar:**  
  İstifadəçi girişi, məhsulların göstərilməsi, səbət əməliyyatları, sifariş prosesi, səhv mesajlarının göstərilməsi.

- **Daxil olmayanlar:**  
  Backend API testləri, real ödəniş inteqrasiyaları, performans və yük testləri.

## 4. Test Mühiti
- Əməliyyat sistemləri: Windows, macOS, Android, iOS  
- Brauzerlər: Chrome (son versiya), Firefox, Edge, Safari  
- Alətlər: Postman (API testləri üçün)

## 5. Testin İcrası
- İlk olaraq manual testlər həyata keçiriləcək.  
- Tapılan səhvlər sənədləşdiriləcək və layihə komandası ilə paylaşılacaq.
